# TokenCostBar 设计文档

## 1. 产品定位

TokenCostBar 是一个 macOS 状态栏工具，用来统计本机常见 AI agent 的每日 token 等值花费。

它不是账本，不是订阅管理工具，也不是复杂 BI 报表。它只做一件事：

> 在 Mac 状态栏里，用最少的信息告诉用户：今天 AI agent 大约花了多少钱，以及最近每天的趋势如何。

产品气质应当克制、简约、简洁。前台只展示统计结果，后台负责解析日志、匹配模型价格、计算美元与人民币金额。

## 2. 核心原则

### 2.1 状态栏是主界面

用户大多数时间只需要看状态栏上的一个数字，例如：

```text
$3.42
```

状态栏数字是产品的核心体验。它应该稳定、轻量、低打扰。

### 2.2 后台聪明，前台安静

模型价格、模型名映射、token 类型、汇率等都是计算逻辑，不应该成为用户每天需要理解或配置的内容。

UI 只展示最终统计结果：

- 今日花费
- 按天走势图
- 按 agent 拆分

### 2.3 不做解释型财务指标

产品只做统计，不评价用户是否“回本”或“划算”。

不做：

- 订阅费用按天摊销
- 等值回本倍率
- 项目维度统计
- 会话维度统计
- 模型维度统计
- token 类型展示
- CSV 导出

## 3. 目标用户

主要用户是经常使用 AI coding agent 的开发者，例如：

- Claude Code 用户
- Codex 用户
- Cursor 用户
- 其他会在本地留下 token usage 记录的 agent 用户

用户想快速知道：

- 今天大概用了多少钱
- 最近几天花费有没有明显变高
- 哪个 agent 是主要花费来源

## 4. 核心功能

### 4.1 状态栏金额

状态栏常驻显示今日美元花费：

```text
$3.42
```

默认使用美元，因为官方模型价格以美元计价。

人民币金额只在弹窗中作为估算展示，固定使用：

```text
1 USD = 7 CNY
```

### 4.2 点击弹窗

点击状态栏后显示轻量弹窗。

弹窗只展示三块信息：

```text
Today
$3.42 / ¥23.94

Daily Trend
[按天走势图]

Agents
Claude Code  $2.91
Codex        $0.51
```

设计要求：

- 信息密度适中
- 不展示复杂筛选器
- 不展示 token 明细
- 不展示模型明细
- 不展示项目或会话
- 趋势图优先用细线图或极简柱状图

### 4.3 管理界面

管理界面不是主体验，只用于查看接入状态和更完整的统计。

只保留两个区域：

#### Sources

展示已支持或可接入的 agent：

- Claude Code
- Codex
- Cursor
- 后续可扩展其他 agent

每个 source 显示：

- 名称
- 是否启用
- 是否可读取
- 最后同步时间

不展示模型价格设置，不展示汇率设置。

#### Stats

展示比弹窗稍完整的统计：

- 今日金额
- 本周金额
- 本月金额
- 按天走势图
- 按 agent 花费排行或占比

Stats 页面仍然要保持简洁，不做复杂分析面板。

## 5. 计算逻辑

### 5.1 数据来源

App 从本机 agent 日志中读取 token usage 信息。

典型字段包括：

- agent 类型
- 使用时间
- 模型名
- input tokens
- output tokens
- cached input tokens，如果日志中存在

这些字段用于后台计算，但不会全部展示在 UI 上。

### 5.2 内置价格表

官方价格表和模型名映射写死在 App 内。

例如内部可以维护：

```text
model aliases -> model price
```

示例：

```text
claude-sonnet-4-5
claude-sonnet-4.5
claude-sonnet
→ Claude Sonnet price

gpt-5-codex
gpt-codex
codex
→ Codex price
```

官方价格变更时，通过发布新版 App 维护。

MVP 不做：

- 自动拉取官方价格表
- 用户手动编辑价格
- 价格版本管理界面

### 5.3 固定人民币估算

人民币金额只做估算，统一使用固定汇率：

```text
CNY = USD × 7
```

MVP 不做实时汇率，不做汇率配置界面。

### 5.4 未知模型

如果日志中出现无法识别的模型：

- 该部分 usage 可先不计入金额
- 不弹出打扰用户的错误
- 可以在 Sources 或 Stats 中显示一个轻量提示

示例：

```text
Some usage could not be priced
```

后续通过新版 App 补充模型映射。

## 6. 界面设计

### 6.1 视觉风格

整体风格应接近 macOS 原生系统工具：

- 低饱和
- 少颜色
- 少装饰
- 不做营销式大标题
- 不做卡片堆叠
- 不做复杂图表

状态栏和弹窗应像系统自带工具一样安静。

### 6.2 状态栏

状态栏只显示今日美元金额。

金额为 0 时：

```text
$0.00
```

金额较大时保持简短：

```text
$128
```

可选后续能力：

- 用户选择状态栏显示 USD 或 CNY
- 用户选择隐藏货币符号，仅显示数字

MVP 默认只显示 USD。

### 6.3 弹窗布局

弹窗宽度控制在较小范围，例如 360-420px。

推荐结构：

```text
┌─────────────────────────┐
│ Today                   │
│ $3.42        ¥23.94      │
│                         │
│ Daily Trend             │
│ ─╮ ╭─╮                  │
│  ╰─╯ ╰──                │
│                         │
│ Agents                  │
│ Claude Code     $2.91   │
│ Codex           $0.51   │
└─────────────────────────┘
```

弹窗底部可以有两个轻量按钮：

- Refresh
- Manage

不需要更多操作。

### 6.4 管理界面布局

管理界面使用两个 tab 或左侧两项导航：

- Sources
- Stats

#### Sources 示例

```text
Sources

Claude Code    Enabled    Ready       Synced 10:42
Codex          Enabled    Ready       Synced 10:42
Cursor         Disabled   Not set up  -
```

#### Stats 示例

```text
Stats

Today     $3.42 / ¥23.94
This Week $18.20 / ¥127.40
This Month $64.10 / ¥448.70

Daily Trend
[更大的按天走势图]

Agents
Claude Code  $51.20
Codex        $12.90
```

## 7. 数据存储

本地存储即可，推荐 SQLite。

需要保存：

- 已解析 usage event
- 每日聚合结果
- agent source 状态
- 最后扫描位置
- 未识别模型记录

不需要上传云端。

隐私原则：

- 默认本地处理
- 不上传原始 prompt
- 尽量只保存必要统计字段
- 如果需要保存原始日志引用，只保存文件路径和位置，不保存内容

## 8. MVP 范围

第一版只需要做到：

- macOS 状态栏常驻金额
- 点击弹窗显示 Today、Daily Trend、Agents
- 管理界面显示 Sources、Stats
- 支持 Claude Code
- 支持 Codex
- 内置模型价格表
- 固定 USD/CNY = 7
- 本地 SQLite 聚合

暂不做：

- Cursor 支持
- 实时汇率
- 自动更新价格表
- CSV 导出
- 项目统计
- 会话统计
- 模型统计
- token 类型展示
- 订阅摊销
- 回本倍率

## 9. 后续扩展

后续扩展必须遵守“克制、简约、简洁”的原则。

可以考虑：

- 增加更多 agent source
- 增加状态栏显示 CNY 的选项
- 增加未知模型提示
- 增加后台自动刷新频率设置
- 增加月度简单汇总

不建议扩展成复杂成本管理平台。

## 10. 一句话总结

TokenCostBar 是一个极简 macOS 状态栏工具，用内置模型价格和固定汇率，把本机 AI agent 的 token usage 转成每日花费趋势。

它的前台只显示统计结果，后台处理复杂计算。用户看到的永远应该是清楚、安静、足够少的信息。
